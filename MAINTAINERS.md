# Maintainers

This document lists the maintainers of nexcage and defines contact and decision rules.

- Primary Maintainer: @cageforge (architecture, releases)
- Core Maintainer: @moriarti (CI/CD, infra, self-hosted runner)

## Rules
- Two-LGTM for non-trivial changes (>= 100 LOC or ADR-impact)
- ADR required for architectural changes; link the ADR in PR description
- Maintainers triage issues and labels; release tags by Primary Maintainer
