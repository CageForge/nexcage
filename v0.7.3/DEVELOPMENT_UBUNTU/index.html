    Development Environment Setup on Ubuntu¶

    üöÄ Development Environment for Proxmox LXCRI¶

    While Proxmox LXCRI is designed exclusively for Debian Linux and Proxmox VE in production, developers can use Ubuntu as a development environment for code editing, compilation, and unit testing.

    ‚ö†Ô∏è Important: Full integration testing requires a real Proxmox VE server. Ubuntu is only for development convenience.

    Prerequisites¶

    System Requirements¶

    • Ubuntu 22.04 LTS or Ubuntu 24.04 LTS
    • 8GB+ RAM (for comfortable development)
    • 20GB+ free disk space
    • Internet connection for dependencies

    Verify Ubuntu Environment¶

    # Check Ubuntu version
lsb_release -a

# Verify system resources
free -h
df -h /

    Development Dependencies Installation¶

    Install Core Dependencies¶

    # Update package list
sudo apt update

# Install build essentials
sudo apt install -y \
    build-essential \
    git \
    curl \
    wget \
    pkg-config \
    libseccomp-dev \
    libsystemd-dev \
    zfsutils-linux

# Install container runtime dependencies (for testing)
sudo apt install -y \
    crun \
    runc \
    uidmap \
    systemd-container

    Install Zig Compiler¶

    # Download and install Zig 0.15.1
ZIG_VERSION="0.15.1"
ZIG_ARCH="x86_64"  # or "aarch64" for ARM64

curl -L "https://ziglang.org/download/${ZIG_VERSION}/zig-linux-${ZIG_ARCH}-${ZIG_VERSION}.tar.xz" \
    | sudo tar -xJ -C /usr/local --strip-components=1

# Verify installation
zig version

    Install Development Tools¶

    # Install debugging and profiling tools
sudo apt install -y \
    gdb \
    valgrind \
    strace \
    htop \
    tree

# Install VS Code (optional)
wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -o root -g root -m 644 packages.microsoft.gpg /etc/apt/trusted.gpg.d/
sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/trusted.gpg.d/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
sudo apt update
sudo apt install code

    Project Setup¶

    Clone Repository¶

    # Clone the project
git clone https://github.com/cageforge/nexcage.git
cd nexcage

# Verify project structure
tree -L 2

    Configure Git (if not already done)¶

    git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

    Development Workflow¶

    Build Project¶

    # Debug build for development
zig build -Doptimize=Debug

# Release build (production-like)
zig build -Doptimize=ReleaseSafe

# Check build artifacts
ls -la zig-out/bin/

    Run Unit Tests¶

    # Run all unit tests
zig build test

# Run specific test module
zig build test -- --filter "memory"

# Run tests with verbose output
zig build test -- --verbose

    Code Quality Checks¶

    # Format code
zig fmt src/

# Static analysis
zig build analyze

# Check for memory leaks (basic)
zig build test-memory

    Development Configuration¶

    # Create development configuration
cat > config/development.json << 'EOF'
{
  "runtime": {
    "primary": "crun",
    "fallback": "runc",
    "data_dir": "/tmp/nexcage-dev"
  },
  "logging": {
    "level": "debug",
    "format": "text",
    "file": "/tmp/nexcage-dev.log"
  },
  "development": {
    "enable_profiling": true,
    "mock_proxmox": true,
    "skip_zfs_checks": true
  }
}
EOF

    Development Testing¶

    # Test basic functionality (without Proxmox VE)
export PROXMOX_LXCRI_CONFIG="$(pwd)/config/development.json"
./zig-out/bin/nexcage --version
./zig-out/bin/nexcage spec

# Run development smoke tests
zig build test-dev

    IDE Setup¶

    VS Code Configuration¶

    Create .vscode/settings.json:

    {
    "zig.initialSetupDone": true,
    "zig.zigPath": "/usr/local/bin/zig",
    "zig.buildOnSave": true,
    "zig.enableLanguageServer": true,
    "files.associations": {
        "*.zig": "zig"
    },
    "editor.formatOnSave": true,
    "[zig]": {
        "editor.defaultFormatter": "ziglang.vscode-zig"
    }
}

    Create .vscode/tasks.json:

    {
    "version": "2.0.0",
    "tasks": [
        {
            "label": "zig build",
            "type": "shell",
            "command": "zig",
            "args": ["build"],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "presentation": {
                "clear": true
            },
            "problemMatcher": []
        },
        {
            "label": "zig test",
            "type": "shell",
            "command": "zig",
            "args": ["build", "test"],
            "group": "test",
            "presentation": {
                "clear": true
            }
        }
    ]
}

    code --install-extension ziglang.vscode-zig

    Debugging¶

    Debug Build¶

    # Build with debug symbols
zig build -Doptimize=Debug

# Run with GDB
gdb ./zig-out/bin/nexcage

    Memory Analysis¶

    # Check for memory leaks with Valgrind
valgrind --leak-check=full --track-origins=yes \
    ./zig-out/bin/nexcage spec

    System Call Tracing¶

    # Trace system calls
strace -o trace.log ./zig-out/bin/nexcage --version

# Analyze trace
less trace.log

    Production Testing¶

    Proxmox VE Test Environment¶

    For complete testing, you need:

    1. Proxmox VE Server (version 8.0+)
    2. Debian 12 (Bookworm) base system
    3. ZFS storage pool configured
    4. Network configuration for containers

    Setting up Proxmox VE Test¶

    # On actual Proxmox VE server
# Transfer built binary from Ubuntu development
scp zig-out/bin/nexcage root@proxmox-server:/tmp/

# SSH to Proxmox VE server
ssh root@proxmox-server

# Install and test on Proxmox VE
cp /tmp/nexcage /usr/local/bin/
chmod +x /usr/local/bin/nexcage

# Run full integration tests
nexcage --version
nexcage create test-container --image alpine:latest
