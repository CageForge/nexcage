    Nexcage Onboarding Guide¶

    Prerequisites¶

    Before starting, ensure you have: - A Linux system (Ubuntu/Debian or RHEL/CentOS/Fedora) - Root access - At least 10GB of free disk space - A disk or partition for ZFS pool

    Quick Start¶

    For a quick setup, run the bootstrap script:

    # Download and run bootstrap script
curl -O https://raw.githubusercontent.com/cageforge/nexcage/main/scripts/bootstrap.sh
chmod +x bootstrap.sh
sudo ./bootstrap.sh

    Manual Setup¶

    1. System Requirements¶

    Ubuntu/Debian¶

    sudo apt-get update
sudo apt-get install -y \
    build-essential \
    git \
    curl \
    wget \
    zfsutils-linux \
    libzfs-dev \
    libproxmox-backup-qemu0-dev \
    gdb \
    strace \
    tcpdump \
    valgrind \
    linux-tools-common \
    linux-tools-generic

    RHEL/CentOS/Fedora¶

    sudo dnf install -y \
    gcc \
    gcc-c++ \
    make \
    git \
    curl \
    wget \
    zfs \
    libzfs \
    gdb \
    strace \
    tcpdump \
    valgrind \
    perf

    2. Install Zig¶

    # Download Zig
ZIG_VERSION="0.15.1"
ZIG_DIR="/opt/zig"
sudo mkdir -p "$ZIG_DIR"
cd "$ZIG_DIR"
sudo wget "https://ziglang.org/download/$ZIG_VERSION/zig-linux-x86_64-$ZIG_VERSION.tar.xz"
sudo tar -xf "zig-linux-x86_64-$ZIG_VERSION.tar.xz" --strip-components=1
sudo rm "zig-linux-x86_64-$ZIG_VERSION.tar.xz"

# Add to PATH
echo "export PATH=\$PATH:$ZIG_DIR" >> ~/.bashrc
source ~/.bashrc

    3. Clone Repository¶

    REPO_DIR="/opt/nexcage"
sudo mkdir -p "$REPO_DIR"
sudo chown $USER:$USER "$REPO_DIR"
git clone https://github.com/your-org/nexcage.git "$REPO_DIR"
cd "$REPO_DIR"

    4. Build Project¶

    zig build

    5. Configure ZFS¶

    # Create ZFS pool
sudo zpool create nexcage /dev/your-disk

# Verify pool
sudo zpool status

    6. Configure Runtime¶

    # Create config directory
sudo mkdir -p /etc/nexcage
sudo cp config.example.json /etc/nexcage/config.json

# Create runtime directory
sudo mkdir -p /run/nexcage

    7. Start Service¶

    # Create systemd service
sudo tee /etc/systemd/system/nexcage.service << EOF
[Unit]
Description=Proxmox LXCRI Container Runtime
After=network.target

[Service]
Type=simple
ExecStart=/opt/nexcage/zig-out/bin/nexcage --config /etc/nexcage/config.json
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# Reload and start service
sudo systemctl daemon-reload
sudo systemctl start nexcage
sudo systemctl enable nexcage

    Development Environment¶

    1. IDE Setup¶

    VS Code¶

    1. Install VS Code
    2. Install Zig extension
    3. Configure settings:
    {
    "zig.path": "/opt/zig/zig",
    "zig.buildRunnerPath": "/opt/zig/zig",
    "zig.formattingProvider": "zigfmt"
}

    2. Debugging Setup¶

    GDB Configuration¶

    # Create .gdbinit
echo "set auto-load safe-path /" > ~/.gdbinit

    Debug Build¶

    zig build -Doptimize=Debug

    3. Testing Environment¶

    # Run tests
zig build test

# Run specific test
zig build test --test-filter "HookExecutor"

# Run integration tests
zig build test_integration

    Common Issues¶

    1. ZFS Pool Creation¶

    # Check available disks
lsblk

# Create pool with specific disk
sudo zpool create nexcage /dev/sdX

    2. Build Errors¶

    # Clean build
rm -rf zig-cache zig-out
zig build

    3. Service Issues¶

    # Check service status
sudo systemctl status nexcage

# Check logs
sudo journalctl -u nexcage -f
